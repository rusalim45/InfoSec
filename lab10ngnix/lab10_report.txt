Task: Explore Nginx proxy_pass parameter to allow sending requests to a running API.

Implementation:

A simple Flask API was created (app.py) running on port 5000 with the endpoint /hello returning “Hello from Flask”.

The application was started using:

python app.py

The backend was tested directly:

$ curl http://127.0.0.1:5000/hello

Hello from Flask

Then NGINX was configured to use proxy_pass:

server {
listen 8000;

location / {
    proxy_pass http://127.0.0.1:5000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}

}

After reloading NGINX:

sudo nginx -t
sudo systemctl reload nginx

The reverse proxy was tested:

$ curl http://127.0.0.1:8000/hello

Hello from Flask

Result:

The request sent to port 8000 was successfully forwarded to the Flask backend on port 5000.

Conclusion:

The proxy_pass directive in NGINX correctly forwards incoming HTTP requests to a backend application server.
